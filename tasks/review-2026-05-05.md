## Problems found — desktop and code

### 🚨 P0 (new) — Contact form is broken
File: `pages/contact.html` lines 138–190.

Every `<input>`, `<select>`, and `<textarea>` is missing both `name` and `id`. The form's `action` is itself (`pages/contact.html`) with `method="get"`, so submission appends nothing useful and reloads to the same page. Visiting the form, filling it out, and clicking submit gives the user a blank confirmation but **no message ever reaches you**.

Fix: add `name="firstName" id="firstName"` etc. to every field, decide whether to wire to Formspree / a Google Form / a serverless endpoint, and add a JS handler that confirms submission. Until that's done, every "Book a Repair" CTA on the site is a dead end on the contact form path. (The `tel:` links still work.)

### Articles have no Open Graph image
A grep for `og:image` across `articles/` returns zero matches across all 33 articles. The seo-content.md spec requires it. Effect: shared article URLs render as bare text on Slack/Twitter/LinkedIn instead of as a card with the article hero image. Quick fix: a one-line `og:image` injection in `scripts/add-seo-improvements.js` and rerun against all articles.

- Meta description: 194 chars — Google cuts at ~160. The last ~30 chars vanish in SERPs.
- Only 3 schemas (`LocalBusiness`, `BreadcrumbList`, `FAQPage`) — `Service` schema would help.

Worth a P2 cleanup pass on the existing 6 city pages before `/seo-hub` adds 22 more identical-shape pages. Otherwise you'll have 28 pages that all need the same fix.

### Other small things
- **Homepage `<h1>` doesn't lead with the primary keyword.** It says `Universal Appliances Repair: Your Friendly Appliance Repair Hub in OC California`. The `<title>` does the right thing (`Appliance Repair Orange County CA | …`) but H1 is heavily weighted by Google. Consider `Same-Day Appliance Repair in Orange County, CA` as the H1 with company name moved to a subheading.
- **Schema completeness on homepage.** Has LocalBusiness + FAQPage + BreadcrumbList. Could also add `Organization` (with `sameAs` for any social links) and an `AggregateRating` once you have Google review counts wired up. P3-1 is already on the plan.
- **H1 mobile clipping at narrow widths.** The hero h1 uses inline `<style>` in `index.html`, not `shared.css`. The 480px breakpoint in shared.css targets `.page-hero h1` not the homepage `.hero h1`, so on a real iPhone the homepage h1 stays at 32px while the rest of the site shrinks to 26px. Add a 480px override inline.
- **`logs/CONTENT_LOG.md` and `logs/AUDIT_LOG.md` don't exist.** `CLAUDE.md` references them, only `HUB_LOG.md` is on disk. The Mon/Wed/Fri scheduled `/seo-blog` and the quarterly `/seo-audit` jobs will both fail to log on first run. Touch the files now (empty headers are fine).
